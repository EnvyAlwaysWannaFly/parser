# Парсер мобильного приложения «Лента»

Скрипт предназначен для сбора данных о товарах и ценах из закрытого API мобильного приложения Android сети гипермаркетов «Лента». 
Реализован сбор данных по двум регионам (Москва и Санкт-Петербург) с аналитикой расхождения цен.

## 🚀 Инструкция по запуску

**ВАЖНО:** API Ленты защищено системой Qrator. Токены сессии (`SessionToken` и `Qrator-Token`) имеют ограниченный срок жизни. 
Если при запуске скрипт возвращает пустой результат или ошибку `403/401`, пожалуйста, обновите заголовки в словарях `headers_msk` и `headers_spb` в `main.py`.

**Как получить актуальные токены:**
1. Откройте приложение "Лента" на Android (или в эмуляторе).
2. Настройте перехват трафика (например, через PCAPdroid, HTTP Toolkit или Charles).
3. Перейдите в каталог товаров нужного города.
4. Скопируйте из запроса `POST /v1/catalog/items` актуальные значения.

## 🛠 Технические вызовы и их решения

В процессе реверс-инжиниринга API были выявлены и успешно решены следующие проблемы:

1. **Гео-зависимое ценообразование и привязка сессий:**
   Изначально выяснилось, что цены не меняются при программной смене магазина через эндпоинты `POST /delivery/mode/set` и `PUT /stores/pickup/{id}`.
   * **Анализ:** Бэкенд Ленты жестко привязывает `SessionToken` к конкретному городу на сервере. При попытке сменить город с текущим `SessionToken` алгоритм Qrator выдавал 403, так как подпись запроса не совпадала.
   * **Решение:** Использование архитектуры изолированных сессий. Для каждого города предварительно генерируется уникальный `SessionToken` (путем сброса данных приложения в эмуляторе). 

2. **Защита целостности данных (Strict Validation):**
   * Внедрена жесткая проверка консистентности. Скрипт анализирует поле `storeId` внутри JSON-ответа первого товара на странице. Если API по ошибке отдает цены чужого региона (например, из-за протухшей сессии), скрипт прерывает работу, чтобы не допустить записи мусорных данных.

3. **Извлечение брендов:**
   API не отдает поле `brand` напрямую в объекте товара.
   * **Решение:** Скрипт парсит справочник фильтров `multicheckbox` -> `brand`, вытаскивает список брендов и производит умный матчинг с названием товара.
